# Body and Clothing Morph (BG3)

A Baldur's Gate 3 mod that lets the player pick a body shape **per character** — **Vanilla**, **SBBF**, or **BCB** — and automatically shows correctly fitted clothing and armor for that body. Selection happens at runtime (in-game, per character, saved in the savegame); no global overwrite of vanilla files, so vanilla, SBBF, and BCB characters can stand side by side in one party.

**Status: pre-release / in development (2026-07-11).** Working in local builds: per-character body selection via MCM UI or console, ~340 vanilla garments refitted per body, cloth simulation (draping skirts/tabards) preserved on refit armor, per-character persistence across save/load.

## How it works

Two paks:

- **ClothMorphContent.pak** (content, large — not in this repo): refit VisualBank resources + refit GR2 meshes for each supported body, generated by the pipeline below.
- **ClothMorphRuntime.pak** (runtime, small — source in `runtime/`): a BG3 Script Extender Lua mod. Per character it applies the chosen nude body via a CharacterCreationSharedVisual override and swaps worn-gear visuals to the matching refit set via an EquipmentRace-keyed map. Choices persist in `PersistentVars`. Configuration UI via **Mod Configuration Menu** (hard dependency).

### The "Path B" mesh pipeline (the part that makes this generalize)

Refit meshes are produced **without ever letting a converter rebuild the mesh container**:

1. `Divine.exe` GR2→GR2 rewrite of the source garment (verified to preserve vertex order exactly, yielding an uncompressed container).
2. A positions-only binary patch (`pipeline/patch_gr2_positions.py`) writes the deformed rigid-submesh positions into that container. Cloth-sim submeshes and all other vertex data stay byte-identical to source.
3. An offline gate re-exports the patched GR2 and verifies byte-equality with the source on every attribute except the intended rigid positions.

This matters because GLB round-trips permute GR2 vertex order, which scrambles BG3's index-based `ClothProxyMapping` buffers and makes cloth simulation explode. Keeping the source container intact keeps the original cloth data valid — so refit armor still drapes. Full forensic write-up in `docs/Cloth Explosion Root Cause Review 2026-07-10.md`.

Body deformation itself is a body-to-body displacement transfer (`pipeline/deform_improved.py`): k-nearest displacement field from the vanilla body to the target body, Gaussian falloff, plus a clearance/clipping pass.

## Third-party assets & permissions

This repo contains **no third-party or Larian assets** — code, configuration, and documentation only. The content pak is built locally from the user's own game files and (where permitted) installed body-mod assets:

- **SBBF** by FredKarrera (Nexus 4864) and **SBBF Vanilla Outfits Patch** (Nexus 5040) — used per their public asset-use terms, with credit.
- **BCB** by Sindae (Nexus 2351) — direct permission granted 2026-06-21; BCB is a required download, never bundled.
- Other outfit mods are only processed with the author's opt-in: see `docs/Modder_Permission_Flag_v1.md` for the machine-readable `ClothMorphPermission.json` flag the build pipeline honors.
